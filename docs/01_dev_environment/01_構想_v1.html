1. 現状と課題

まず、私たちが直面している一般的な開発の課題を整理します。これらの課題を解決することが、新しいワークフローの目的です。

現状

  • Python (Flask) でWebアプリケーションを開発
  • WebARENA上にテスト環境と本番環境を保有

従来の課題

  • コーディング、デバッグ、テストに時間がかかる
  • デプロイが手動で手間がかかる
  • ドキュメント作成が非効率
  • AIに都度背景を説明する必要がある

2. 目指す理想のワークフロー

これらの課題を解決するため、「楽して開発」を実現する自動化されたワークフローを目指します。AIを単なるツールではなく、開発プロセス全体を理解するパートナーとして位置づけます。

3. 開発フェーズ

AIによる自動化とプロジェクト全体のコンテキスト保持を中心に、コーディング作業を劇的に効率化します。

使用ツール

💻
VS Code
🚀
Roo Code
🔄
OpenRouter
🌿
Git / GitHub

プロセス

  1. VS Code上でRoo Codeに自然言語でタスクを指示します。
  2. Roo CodeがOpenRouter経由で最適なAIモデルを選択し、コードを自動生成します。
  3. プロジェクトのコンテキスト（ファイル構造、過去の変更履歴など）をRoo Codeが自動で保持します。
  4. GitHubでソースコードと、デプロイなどの自動化設定ファイルを一元管理します。

4. テスト & デプロイフェーズ

継続的インテグレーション(CI)と継続的デプロイメント(CD)の考え方を取り入れ、リリースプロセスを自動化・高速化します。

使用ツール

🚀
Roo Code
🌿
Git / GitHub
☁️
WebARENA
🔑
SSH

プロセス

  1. 開発完了後、Roo Codeに「テスト環境にデプロイして」と指示します。
  2. Roo Codeが自動でGitにコミットし、SSH経由でWebARENA上のテストサーバーに資材をアップロードします。
  3. 動作確認後、「本番環境に資材をリリースして」と指示します。
  4. Gitにリリース用のタグが付けられ、SSH経由で本番サーバーに最終資材が安全にデプロイされます。

5. 運用 & 環境構築

リリース後のドキュメント作成やログ管理も自動化し、次の開発サイクルに備えます。まずは最初のステップから始めましょう。

運用コンセプト

  • Gitのコミット履歴やタグ情報から、Roo Codeがリリースノートを自動生成します。
  • サーバーログをRoo Codeに分析させ、異常検知やパフォーマンス改善に役立てます。

環境構築の最初のステップ

  1. VS CodeにRoo Code拡張機能をインストールします。
  2. OpenRouterでアカウントを作成し、APIキーを取得します。
  3. Roo Codeの設定ファイルに、APIキーとWebARENAへのSSH接続情報を記述します。
